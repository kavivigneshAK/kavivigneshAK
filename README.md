<details>
<summary><b>🤖 SkillBridgeAI — AI Employability & Interview Preparation Platform</b> &nbsp;|&nbsp; Feb 2026</summary>

<br/>

> An end-to-end AI-powered employability platform designed to bridge the gap between candidate skills and industry requirements through intelligent resume analysis, automated skill-gap detection, and real-time AI-driven interview simulation.

<div align="center">

| Attribute | Details |
|:---|:---|
| **Stack** | React · FastAPI · TensorFlow.js · COCO-SSD · Python · MongoDB |
| **Core AI** | Resume NLP parser · Skill-gap detection model · Multilingual interview bot |
| **Scale** | Multi-user concurrent sessions · Real-time proctoring pipeline |
| **Performance** | Sub-200ms resume analysis · Real-time video inference at 15 FPS |
| **Security** | JWT-based auth · Encrypted session management |
| **Impact** | Finalist — PALSInnowah Hackathon 2025 Cluster |
| **Repository** | [![GitHub](https://img.shields.io/badge/View%20Repo-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/kavivignesh-ak/skillbridge-ai) |

</div>

**Engineering Overview:**
SkillBridgeAI is architected as a three-layer intelligence platform. The **resume intelligence layer** ingests PDF/DOCX resumes and uses NLP-based parsing to extract skills, experience, and education, then cross-references against a curated job-role skill taxonomy to produce a structured skill-gap report. The **interview simulation layer** deploys a multilingual AI interview bot built on FastAPI, capable of generating domain-specific questions and evaluating candidate responses with contextual scoring. The **proctoring layer** integrates TensorFlow.js with COCO-SSD to perform real-time browser-based object detection, flagging prohibited items or suspicious behavior during sessions — all without server-side video streaming, ensuring zero latency and full privacy compliance.

<br/>

</details>

The **JWT authentication system** implements access and refresh token rotation with protected middleware chains. <details>
<summary><b>📊 AI/ML Error Mitigation Dashboard — Model Explainability Platform</b> &nbsp;|&nbsp; Sep 2025</summary>

<br/>

> An interactive machine learning observability dashboard that benchmarks 5+ classification and regression models, surfaces prediction error patterns using ensemble analysis, and delivers feature-level explainability through SHAP visualizations.

<div align="center">

| Attribute | Details |
|:---|:---|
| **Stack** | Python · Streamlit · XGBoost · LightGBM · CatBoost · SHAP · Plotly |
| **Models** | XGBoost · LightGBM · CatBoost · RandomForest · GradientBoosting |
| **Explainability** | SHAP TreeExplainer · Force plots · Summary plots · Dependency plots |
| **Scale** | 5+ models benchmarked simultaneously · Dataset-agnostic pipeline |
| **Performance** | Real-time SHAP computation · Interactive Plotly rendering |
| **Impact** | Production-ready ML observability tool for model auditing |
| **Repository** | [![GitHub](https://img.shields.io/badge/View%20Repo-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/kavivignesh-ak/ml-error-dashboard) |

</div>

**Engineering Overview:**
The AI/ML Error Mitigation Dashboard addresses the interpretability gap in production ML systems. The platform accepts any structured tabular dataset, runs it through a benchmarking pipeline across 5 ensemble models with unified preprocessing, and surfaces comparative performance metrics (AUC, F1, RMSE, MAE) on a Streamlit dashboard with side-by-side charting. The **SHAP integration** uses TreeExplainer to generate global feature importance rankings and local per-prediction force plots, enabling engineers to diagnose systematic prediction failures at the feature level. The **Plotly visualization layer** produces interactive residual distribution plots, learning curves, and confusion matrices — enabling non-technical stakeholders to audit model behavior without code.
